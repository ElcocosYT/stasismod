# Contributing to Stasis Paradox

Thanks for your interest in contributing.

Contributions are welcome through:

- Pull requests from forks
- Suggested patches
- Bug reports
- Balance feedback
- Documentation improvements

## Contribution workflow

1. Fork the repository on GitHub.
2. Create a branch for your change.
3. Make your changes as cleanly as possible.
4. Open a pull request with a clear description of what changed and why.

Please keep pull requests focused. Small, well-scoped changes are much easier to review and merge.

## What you may and may not do

You may:

- Fork this repository on GitHub for the purpose of proposing changes
- Modify your fork locally or on GitHub while preparing a pull request
- Share code snippets in issues or pull requests as part of contributing

You may not, unless you have prior written permission from the maintainer:

- Publish this mod as your own project
- Redistribute compiled builds of this mod
- Reupload this repository or its contents to other sites
- Publish modified versions of this mod
- Create public releases based on this project outside the official project pages

These restrictions apply even if you created a fork for contribution purposes. A fork used to contribute is not permission to publish a separate release of the mod.

Accepted contributors may be credited in one or more of the following places:

- Git commit history
- Pull request history
- Release notes or changelog

Credit is appreciated, but merge decisions and contributor listings remain at maintainer discretion.
